## 示例 1：技术硬核 (Technical)

### 写作素材
```
主题：深入理解 Vue 3 响应式系统原理

核心要点：
- Vue 3 使用 Proxy 替代 Vue 2 的 Object.defineProperty
- reactive() 和 ref() 的实现原理和区别
- 依赖收集机制：effect、track、trigger
- 响应式系统的性能优化策略
- 与 Vue 2 响应式系统的对比

技术细节：
- Proxy 可以拦截对象的所有操作，包括属性的添加和删除
- ref() 通过 .value 访问，内部使用 RefImpl 类实现
- WeakMap 存储依赖关系，避免内存泄漏
- computed 的懒执行和缓存机制

代码示例需求：
- 简单的响应式对象创建
- 依赖收集的核心代码片段
- effect 函数的基本用法
```

**推荐风格**：技术硬核 (Technical)

## 示例 4：学术严谨 (Academic)

### 写作素材
```
主题：前端框架响应式系统的设计模式研究

研究背景：
- 现代前端框架普遍采用响应式编程范式
- 不同框架的响应式实现各有特点
- 响应式系统对应用性能和开发体验有重要影响

研究对象：
- Vue.js 的响应式系统（Proxy-based）
- React 的状态管理（Hooks + Fiber）
- Svelte 的编译时响应式
- Solid.js 的细粒度响应式

研究方法：
1. 文献综述：分析各框架官方文档和技术论文
2. 源码分析：深入研究核心实现代码
3. 性能测试：对比不同场景下的性能表现
4. 案例研究：分析实际项目中的应用效果

理论框架：
- 观察者模式（Observer Pattern）
- 发布-订阅模式（Pub-Sub Pattern）
- 依赖注入（Dependency Injection）
- 惰性求值（Lazy Evaluation）

研究发现：
1. Proxy-based 方案提供更完整的拦截能力
2. 编译时优化可显著减少运行时开销
3. 细粒度更新策略在大型应用中优势明显
4. 不同方案在内存占用和更新效率上存在权衡

结论与展望：
- 响应式系统设计需要平衡性能、开发体验和学习曲线
- 未来趋势：编译时优化 + 运行时响应式的混合方案
- 建议：根据项目规模和团队技术栈选择合适的框架

参考文献：
[列出相关技术文档、论文、博客文章]
```

**推荐风格**：学术严谨 (Academic)

## 示例 5：轻松随笔 (Casual)

### 写作素材
```
主题：程序员的日常吐槽：那些年我们踩过的坑

吐槽点：
1. 命名的艺术
   - 变量命名永远是最难的事
   - data、data1、data2、newData、finalData...
   - 看到 temp、test、aaa 就知道是临时代码，结果用了三年

2. 注释的玄学
   - "这段代码很简单，不需要注释"
   - 三个月后：这 TM 谁写的？
   - 一查 Git：原来是我自己...

3. Bug 的诡异
   - 明明本地运行正常，一上线就出问题
   - 删掉一行无关代码，Bug 神奇地消失了
   - 最后发现是缓存没清...

4. 需求的变化
   - 产品：这个功能很简单，就改个颜色
   - 实际：涉及 10 个文件，3 个接口，2 天工作量
   - 产品：那顺便把这个也改了吧...

5. 技术选型的纠结
   - 新项目用什么框架？Vue？React？还是 Svelte？
   - 纠结一周，最后还是用最熟悉的
   - 然后看到新框架发布，又开始纠结...

6. 加班的日常
   - "这个需求很急，今晚能搞定吗？"
   - "明天上线，今天发现个 Bug..."
   - "周末加个班，下周给你调休"（然后就没有然后了）

心得体会：
- 编程不易，且行且珍惜
- 保持学习，但不要焦虑
- 代码规范很重要，未来的自己会感谢你
- 适当吐槽，保持心态平和
- 记住：Bug 是修不完的，但生活还要继续
```

**推荐风格**：轻松随笔 (Casual)

## 示例 6：技术硬核 - 性能优化

### 写作素材
```
主题：前端性能优化实战：从 3s 到 300ms 的加载优化之路

项目背景：
- 电商平台商品列表页
- 初始加载时间：3.2s
- 目标：首屏加载 < 1s

性能分析：
1. 网络层面
   - 首页资源体积：2.5MB
   - 图片未压缩，平均 500KB/张
   - 未启用 HTTP/2
   - 无 CDN 加速

2. 渲染层面
   - 首屏渲染 1200+ 个 DOM 节点
   - 未使用虚拟滚动
   - 大量重排重绘
   - 动画使用 JavaScript 实现

3. 代码层面
   - Webpack 打包未优化
   - 未做代码分割
   - 第三方库全量引入
   - 无 Tree Shaking

优化方案：
1. 资源优化
   - 图片压缩 + WebP 格式
   - 启用 Gzip/Brotli 压缩
   - 配置 CDN
   - 资源懒加载

2. 渲染优化
   - 虚拟滚动（vue-virtual-scroller）
   - 骨架屏占位
   - CSS 动画替代 JS
   - 防抖节流

3. 构建优化
   - 代码分割（动态 import）
   - Tree Shaking
   - 按需引入第三方库
   - 生产环境移除 console

4. 缓存策略
   - Service Worker
   - 强缓存 + 协商缓存
   - LocalStorage 缓存接口数据

优化结果：
- 首屏加载：3.2s → 850ms
- 资源体积：2.5MB → 600KB
- FCP：2.1s → 400ms
- TTI：3.5s → 1.2s
- Lighthouse 评分：45 → 92

关键代码片段：
- Webpack 配置优化
- 虚拟滚动实现
- 图片懒加载组件
- Service Worker 缓存策略
```

**推荐风格**：技术硬核 (Technical)

## 使用建议

1. **选择合适的风格**：根据目标读者和内容类型选择写作风格
2. **提供充足的素材**：素材越详细，AI 生成的内容越贴合需求
3. **明确结构**：使用标题、列表等结构化方式组织素材
4. **包含关键信息**：技术细节、代码示例、数据支撑等
5. **迭代优化**：生成后可以根据结果调整素材，重新生成

## 快速开始

1. 复制上述任一示例的"写作素材"部分
2. 粘贴到 AI 写作助手的输入框
3. 选择对应的写作风格
4. 点击"生成草稿"按钮
5. 查看生成结果，根据需要进行调整
